API 1: Business Objectives


 The Problem 

We want to make tracks in our database, which are saved in midi format, accessible to our clients as regular audio files, as well as to make sharing these tracks directly with social networks possible.


 The Impact 

Users of our website will be able to share their created tracks freely to other platforms or download them.


 Key User Stories
 
 1. As an artist, I want to have my music in an audio format, so that I can share it.
 2. As a social media user, I want to share a track, so that it’s accessible on my social profile.


  API Description

Description: the MIDI file will be converted to an audio file using an external library

Pros: - Audio formats like MP3 are highly standardized and well supported, and are likely to  be more useful for our user than MIDI format

Cons: - Conversion to MP3 reqires computational power

 Authors

 Problem

Tracks in our Database need to be converted to audio formats to make access to them more                                        accessible.


 Solution

This API converts midi files to regular Audio files, so that they can be acessed with regular media players or external platforms like social media sites.


 Implementation

The corresponding MIDI file will be read from our servers identified by an track-id. 

Conversion will be handled by an external library.

The response will include a MP3 file in the payload.
 
 Authentication
 

No authentication is required, this API is publicly available.

API 2 Business Objectives


 The Problem 

We want to enable link sharing and have permanent links for tracks created and saved using our platform, so that users can show their achievements to their friends on social media.


 The Impact

This API receives a Title of a song and returns a matching track from the db in case it exists.


 Key User Stories


 List several key user stories for your API with the following template:
 As a [user type], I want [action] so that [outcome].

1. As an external content provider, I want to be able to find a track by searching the name

2. As a music listener, I want to be able to find a  track by searching with the name.
  
  API Description


  Description: Unique https links will be generated for saved tracks and are seperated from our the IDs we use internally  

  Pros:  - Security, because internal IDs are not exposed.

Cons: - More proccessing needed and more data needs to be stored in the database

 


 

 

API 2 Specification 


 Title

matcher


 Authors

Zidane, Herzog


 Problem

Songs need to be accessible by means other than the internal ID under which it is saved.


 Solution

Make IDs retrievable by searching the song title.
 

Implementation

Each track has an external ID that is used for permanent links. This ID is required by other APIs and can be retrieved by providing title. With pattern matching, the corresponding track will be looked up and the external ID will be returned to the client if found.
 
 Authentication

No authentication is required, this API is public.
